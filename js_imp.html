Most out of JS

The words I will be writing will be valid for any other programming language but here I will be more focusing on JavaScript.

Let's start with understanding why should we learn any programming language. Suppose I give you a map and ask you to tell the shortest path between two cities A and B which are also connected to other cities and also can be directly connected by roads of different thing i.e simply form a network type of thing. If you were to solve it then you might solve it by any means because we have a big brain ! But now if you want your computer to do this task for you (which exactly Google Maps does) then just think what can you do ??

One thing you can tell your computer that just add all the paths lengths and then select the minimum one. Wait what did I say.. tell ? How can you tell your computer? Computers can't hear just like us. So, actually telling means that you have to write the set of instructions in the computer. But does computer understand English, Latin, German, or what...?

Does computer understand JavaScript, C++, Python, etc.? Yes NO !!

Computer only understand machine level codes which are binaries. So, can you even think of writing even a print statement...oh I cannot. So that is why we have some beautified languages like JavaScript, C++ and all those which when compiled becomes readable by computer. That is simply the programming language.

Now let's come to the problem discussed. There we wrote a instruction of comparing all distances. But now we can again imagine the network as a picture in our mind because of big brains but what about computers. They need a proper way to store the road lengths name of cities linked, etc. and make a proper interrelation. So these creation of proper structures of data are called the data structures. Many data structure exists like arrays, lists, linked lists, trees, heaps, graphs, etc. So, one might ask why so many close types of data structures are needed ...!! It's because of speed every data structure has there own pros and cons... in some insertion time is more then in some searching time is more... that's why choosing correct data structure for problem is very important.

Talking about speed will the process we defined above for solving the problem work well i.e work fast. If network is very small it will do. But increasing the map increases time taken by a huge rate. So, overall we can say its a bad process or a bad algorithm. The time can even increase up to years. And that is the reason development of algorithms is necessary.

Now enough of general talks and let's discuss that what benefits are you gaining with learning JavaScript.

Firstly as I told you can write algorithms and data structures using the language. Also it can be used in web designing with HTML and CSS to provide interactivity to your website which cannot be done with any other programming language. Also it can be used to design the back-end of the website which can be also done with other languages like python.. But you are gaining a huge benofit i the front-end right. Because when any other person who had learned any other programing language will have to again learn JavaScript during web designing. But you are already done ..!!

Also JavaScript is a pre-compiled language for which its execution time is faster than many other programming language.

Proper learning :

One straight forward thing is that if you know any other programming language you know the basics of any language and can easily learn any language. Because all languages have same things, only the style of writing is slightly different. But considering you are learning a language for the first time then there are a very basic syntaxes which you have to learn... like variable assignment, conditional statements (if..else and switch), loops (for and while) and like this. Also another important thing to learn just after that is Object Oriented Programming. Understanding the proper concept behind it is very important. It can be that you will directly not create classes or objects at the beginning but from the beginning only you will be using OOPs unconsciously. Different libraries are what ? They are actually classes! When you create an array actually you are creating an object of the array class. Here I will not go deep into OOPs because its a pretty big topic, but I am advising to understand (not hustling with it for one month and remembering each and everything of it) OOPS.

After each and every smallest doubt has been cleared in this above section you are ready to move on to data structures and algorithms. Here different data structures have some of there own methods (functions of a class are called methods). So, there you will be introduced to some more syntaxes. Like when you learn array. What are the operations that you can do on it ? You can slice it it splice it, etc. So do you have to remember all the syntaxes..? No...suppose you want to slice an array and you forgot what was the syntax, you can just make a simple google search that "how to slice an array" and the full syntax with proper explanation will be clearly displayed. Then what you need to know? You should know how to use that method. The array created will be an object so to apply any method you just have to do arr.slice(1:5). That is simply understand the concepts. You have to be capable of reading any documentation and understand that.

When you will be asked to solve any algorithmic problem you are asked to write the pseudocode first and you are mainly judged on based on that. Because everyone wants to see a proper approach in solving the problem not the language you are solving in. In pseudocode you write just in English that if this equals to this..etc. In pseudocode you have to decide what data structure is to be used for that particular problem for faster computational speed. So that is what I want to focus that conversion of a logic in programming language is not the main thing. Creating the logic is the main thing.

Thus, after learning the basic things I will advice to straight jump into data structures in somewhat following way : lists→sets→array→linked lists→hash tables→trees→heaps→graphs etc. By learning these data structures I mean that understanding all the operations that we can perform on these, time complexities of different operations, in what aspect one data structure is different from another, in which scenario which data structure is best to use.

Next you can dive on different algorithms. First start with simple algorithms like reversing array, reversing linked lists and many more interesting things. The main thing to do to brush up your algorithm skills are to solve problems. You can start solving problems from websites like Hackerrank, Leetcode, Codechef, etc. Then start learning different different algorithms on data structures like traversing a tree, graph, etc. And slowly you will realize that how you are entering and solving real world problems.

Also some important topics like async JavaScript in es6 is very important which you can read once you get comfortable with the basics.

Web Development:

As I told earlier that learning JavaScript you can work with algorithms and data structures but also it is a core of web development. Notice I am using development...it means both front-end and back-end !! Suppose you learn python then you can only work on back-end....but with JavaScript its boom...

Front-end:

Using HTML you build the skeleton of the website. Using CSS you can make your website cool. And using JavaScript you can add interactivity to your website that if you clack on this button this will happen, hover on this image this will happen, make your window size small this will happen and literally anything you want to.

Back-end:

Fetching information from databases and all other related works are done in the back-end. And you have many libraries in JavaScript to work in back-end. Libraries and modules means new syntaxes but very few and very relatable. All modules have their official documentation which you can read and understand the different syntaxes (don't memorize them..documentation is present only to remove the burden of memorization).

So, that's it I will not write much about web development as it is a different topic.

But overall to conclude that JavaScript is a very powerful programming language and can be used in many scenarios. Hope it helped to give a kickstart. Follow the steps without panicking and it will be a really easy work.
